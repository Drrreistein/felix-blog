01

从「从未如此落后」说起

去年发明了 vibe coding 的 Karpathy，为何今年说自己「从未如此落后」？这背后是 AI 编程工具的一次质变。

去年，Andrej Karpathy 在社交媒体上发明了「vibe coding」这个词，引发了广泛讨论。一年后，在 Sequoia AI Ascent 2026 大会上，他带来了一个更令人深思的论断：「我从未感觉自己如此落后于编程。」

Karpathy 回忆道，去年使用 Claude Code 等 Agentic 工具时，它们能写出不错的代码片段，但时不时需要人类介入修改。然而，到了去年12月，一切都变了。

「我注意到，最新的模型产出的代码片段直接就能用。我不断提出更多需求，代码依然完美运行。我已经想不起上次需要纠正 AI 是什么时候了。」Karpathy 描述道，「这种转变非常突然。」

他特别强调，去年大多数人体验 AI 编程的方式还是「ChatGPT 式的辅助工具」，但12月之后，基于 Agent 的连贯工作流真正开始运转了。这次认知升级让他彻底沉浸在无限多的 side project 中，「vibe coding」变成了日常。

「我已经想不起上次需要纠正 AI 是什么时候了，这种转变非常突然。」
— Andrej Karpathy，1:22
「你们需要重新审视 AI 编程，因为从去年12月开始，一切都发生了根本性变化。」
— Andrej Karpathy，1:52
核心观点

AI 编程在去年12月经历了质的飞跃，从「辅助工具」升级为「能独立完成任务的 Agent」。

02

软件开发的三个时代

Karpathy 提出了软件开发的「三段论」：从手写代码，到训练神经网络，再到现在的大语言模型。

Karpathy 在访谈中系统阐述了他对软件开发范式演变的理解。

软件 1.0：人类手写代码，用明确的规则告诉计算机做什么。

软件 2.0：人类不再写代码，而是创建数据集和训练目标，让神经网络通过学习来确定参数。「编程变成了一种数据整理工作。」

软件 3.0：当 GPT 等大语言模型在足够多的任务上训练后，它们实际上成为了一种可编程的计算机。「编程变成了写提示词，上下文窗口成了操控这个解释器的杠杆。」

Karpathy 用两个生动的例子说明这个转变：

第一个是 OpenClaw 安装。传统方式需要写复杂的跨平台 Shell 脚本。但在软件 3.0 时代，你只需要把一段文字复制给 AI Agent，它就能自动分析你的环境、执行安装、调试问题。「真正强大的不再是精确的指令，而是告诉 Agent 要做什么。」

第二个例子更震撼：MenuGenen 是 Karpathy 用传统方式开发的菜单图片生成应用。但当他看到 Gemini 的软件 3.0 版本时彻底震惊——只需把菜单照片发给 Gemini，告诉它「用 Nanobanana 把菜品图片叠加到菜单上」，它就能直接返回一张完美的渲染图。「MenuGenen 这个应用根本不应该存在。」

「编程变成了写提示词，上下文窗口成了操控这个解释器的杠杆。」
— Andrej Karpathy，3:03
「MenuGenen 这个应用根本不应该存在。神经网络能做更多，我们需要重新思考什么才是真正值得构建的。」
— Andrej Karpathy，6:04
核心观点

软件 3.0 不是让旧事物变得更快，而是让原本不可能的新事物变得可能。

03

可验证性：AI 能力的「锯齿」之谜

为何最先进的 AI 能破解代码库，却建议步行去洗车？Karpathy 用「可验证性」框架解释了这个看似矛盾的现象。

访谈中，Karpathy 深入探讨了 AI 能力的「锯齿状」分布问题。

他指出，前沿实验室训练大语言模型时，本质上是在构建巨大的强化学习环境。当模型被给予「可验证奖励」时，它们会在可验证的领域（如数学、编程）突飞猛进，但在难以验证的领域则进展缓慢。

一个经典案例是「strawberry 有几个 r」的问题，以及更荒诞的新例子：「我想去洗车，车在50米外，我应该开车还是走路？」最先进的 Opus 4.7 模型会建议你走路——因为50米确实很近。但与此同时，它能重构10万行代码库、发现零日漏洞。

Karpathy 强调，这种锯齿状分布部分源于训练数据：他曾发现 GPT-3.5 到 GPT-4 的国际象棋能力大幅提升，背后的原因是有人专门向 OpenAI 建议将大量象棋数据加入预训练集。「我们某种程度上受制于实验室的选择——他们碰巧把什么放进了数据混合。」

这意味着：如果你的应用场景不在 AI 的 RL 电路中，就需要考虑微调或定制开发。

「最先进的 Opus 4.7 能重构10万行代码库，却发现不了自己应该开车去洗车。这是疯狂的。」
— Andrej Karpathy，11:38
「我们某种程度上受制于实验室的选择——他们碰巧把什么放进了数据混合。」
— Andrej Karpathy，12:50
核心观点

AI 能力的分布取决于「可验证性」和「实验室关注度」两个维度，选择创业方向时需要考虑这一点。

04

Agentic Engineering：新学科的崛起

Vibe Coding 提高了编程的下限，而 Agentic Engineering 是为了在提高上限的同时，保持专业软件的质量标准。

Karpathy 在访谈中明确区分了两个概念：

Vibe Coding：让每个人都能编程的工具。它提高了编程的「地板」（下限），让任何人都能「vibe」出任何东西。

Agentic Engineering：保持专业软件质量标准的新学科。「你不能因为 vibe coding 就引入安全漏洞。你和以前一样对软件负责，但你能用 AI 更快地完成工作。」

他将 Agent 描述为「带刺的、有点随机的实体」，Agentic Engineering 就是如何协调这些强大但不可靠的实体，在不牺牲质量的前提下提速。

关于 AI 原生编程者的特征，Karpathy 分享了他的观察：

「就像以前工程师会深入配置 Vim 或 VS Code 一样，现在要最大化利用 Claude Code、Devin 等工具。」他建议企业重新设计招聘流程：「不再是出一道算法题，而是让候选人用 AI 构建一个完整的 Twitter 克隆，然后用 10 个 Claude Code 实例试图破解它。」

在 Karpathy 看来，AI 时代真正重要的能力是：品味、判断力、设计思维，以及对底层原理的理解。「你不需要记住 PyTorch 和 NumPy 的 API 细节，但需要理解张量的底层机制，才能避免不必要的内存复制。」

「以前人们谈论 10x 工程师，现在 Agentic Engineer 的能力放大远不止 10 倍。」
— Andrej Karpathy，17:01
「你不需要记住 API 细节，但需要理解底层原理。你做设计，Agent 做填充工作。」
— Andrej Karpathy，21:53
核心观点

Agentic Engineering 是将 AI 能力转化为专业生产力的新学科，需要全新的工作方式和评判标准。

05

动物 vs 幽灵：理解 AI 的心智模型

Karpathy 提出了一个深刻的类比：我们不是在构建「动物」，而是在召唤「幽灵」——被数据和奖励函数塑造的、参差不齐的智能体。

Karpathy 在访谈中分享了他对 AI 本质的心智模型。

他将 AI 描述为「幽灵」而非「动物」：「动物有内在动机，有好奇心和自主性。而 AI 是被数据和奖励函数塑造的统计模拟电路。你对它吼叫，它不会变得更好或更差——这完全不影响它。」

这种理解带来了重要的实践意义：

首先，你需要保持「在场」。「这些 Agent 仍然会犯奇怪的错误。比如 MenuGenen 的支付系统，我的 Agent 试图用 Stripe 和 Google 的邮箱地址交叉关联用户资金——这是完全随意的对应关系。」

其次，你需要设计详细的规格说明。Karpathy 越来越不喜欢「Plan Mode」，更倾向于与 Agent 协作设计一份详尽的文档，然后让 Agent 负责执行细节。

最后，品味和判断力仍然难以被替代。当 AI 生成代码时，「有时候代码不是特别惊艳，有很多复制粘贴和尴尬的抽象，写出来能用但很粗糙。我希望未来的模型能改进这一点，但目前还没有。」

「我们不是在构建动物，而是在召唤幽灵。它们是被数据和奖励函数塑造的统计模拟电路。」
— Andrej Karpathy，24:38
「你不能外包你的理解，但你可以外包你的思考。」
— Andrej Karpathy，28:05
核心观点

理解 AI 是「幽灵」而非「动物」，是有效使用 AI 的前提——接受它的随机性，同时保持人类在判断力上的主导地位。

06

Agent 原生世界的基础设施

未来的世界需要「Agent 原生」的基础设施——不是为人类设计的，而是为 AI Agent 设计的。

Karpathy 在访谈中描绘了他对 Agent 未来的愿景。

他指出，目前几乎所有基础设施都是为人类设计的——文档面向人类阅读，API 需要人类操作。但未来，这些都需要被重新设计。

「我最喜欢的槽点是：为什么人们还在告诉我该怎么做？我什么都不想做。什么是应该复制给 Agent 的东西？」Karpathy 调侃道，「每次我被告知'访问这个 URL'时，我都想叹气。」

他认为未来需要的是「传感器和执行器」架构：AI Agent 通过传感器感知世界，通过执行器作用于世界。「数据结构和 API 需要设计得让 AI 能轻松理解和使用。」

关于 Agent 的未来，Karpathy 预测：「最终，Agent 会代表个人和组织。 我的 Agent 会和你的 Agent 交流，确定我们会议的细节。」

「为什么文档还在告诉我该怎么做？什么时候文档能变成'把这段复制给你的 Agent'？」
— Andrej Karpathy，25:41
核心观点

从「Human-Native」到「Agent-Native」的转变是未来的大趋势，需要重新设计整个互联网基础设施。

07

智能廉价时代，什么值得深入学习

「你可以外包你的思考，但你无法外包你的理解。」在 AI 时代，学习的本质发生了根本变化。

访谈以一个深刻的问题收尾：当 AI 智能变得廉价时，什么仍然值得深入学习？

Karpathy 引用了一条让他深思的推文：「你可以外包你的思考，但你无法外包你的理解。」

他解释道：「信息仍然需要进入我的大脑。我感觉自己成了瓶颈——我需要知道我们在构建什么，为什么值得做，我如何指导 AI Agents。」

这正是 Karpathy 对 LLM 知识库项目如此兴奋的原因：「这是处理信息的工具。当我看到信息的新视角时，我总能获得洞见。这本质上是对固定数据的一系列提示词，生成合成信息。」

他总结道：「这些增强理解力的工具非常有趣。未来也许 AI 能处理理解本身，但目前，人类仍然独特地负责方向性判断。」

「你可以外包你的思考，但你无法外包你的理解。」
— Andrej Karpathy，28:05
「最终，是理解力在约束我们，而工具可以增强理解力。」
— Andrej Karpathy，29:00
核心观点

在 AI 时代，理解力成为最稀缺的能力——知道要构建什么、为什么值得做、如何指导 AI，这仍然是人类独特的职责。
