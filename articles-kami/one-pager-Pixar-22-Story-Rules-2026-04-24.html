Rule #4 · 故事骨架（Story Spine）

✦ 最强大的故事构建公式 ✦

从前有一天 ___。

每天，___。

直到有一天，___。

因为这件事，___。

又因为这件事，___。

最终，___。

适用于：小说、演讲、品牌叙事、产品介绍、创业路演——任何需要说服的场景。

一 · 叙事结构与过程

Rule #3
先完成，再理解
你在写完故事之前，不会真正知道它在讲什么。先写，结尾处再重写——主题往往在结束时才浮现。
Rule #7
先定结局，再写中间
结局是最难的部分，先把它想清楚。故事、提案、路演——先有终点，再逆向规划。
Rule #8
完成比完美重要
好故事是改出来的，不是想出来的。"没有书是真正写完的，只是被放弃罢了。" 先发布，再迭代。
Rule #22
故事的精髓是什么
先问自己：你想说的最简洁版本是什么？知道核心，剩下的都是填充。

二 · 写作工艺

Rule #5
简化 · 聚焦 · 合并
你会觉得在丢失有价值的东西，但它让你自由。约束是创作的引擎——少即是多。
Rule #11
写下来才能修改
停留在脑海里的完美想法永远无法分享。动笔——草稿的价值在于它存在。
Rule #17
没有工作是真正浪费的
如果某段不 work，让它走，它会以另一种形式回来。写过的内容从不会真正浪费。
Rule #18
测试 ≠ 打磨
知道自己何时在真正测试，何时只是在无意义地打磨。反复抛光计划只是拖延的伪装。
Rule #20
改组你不喜欢的
把一部你讨厌的电影拆解——如果是你，会怎么重组？每个不共鸣的作品都是学习素材。

三 · 角色与观众

Rule #1
观众爱努力，胜过爱成功
人们追随的是旅程，不是终点。故事、产品、品牌——聚焦"怎么做"和"为什么做"，而非"是什么"。
Rule #2
作者无关紧要
你作为写手的乐趣与观众的需求完全不同。忘掉你自己，专注交付给观众。
Rule #6
给角色制造对立困境
角色擅长什么、舒适什么？扔给他们完全相反的东西。怎么应对？这是故事的引擎。
Rule #13
给角色观点
被动、可塑的角色在创作时看似可爱，但对观众是毒药。你的观点要让读者表态，不要模棱两可。
Rule #15
换位：你会怎么感受
把自己放进角色的处境。诚实带来可信度，即使是荒谬的情节。
Rule #16
赌注是什么
给读者一个支持角色的理由。如果失败会怎样？把胜算压到最低。
Rule #21
与角色建立认同
不能只写"酷"。要问：是什么让"你"会那样行动？让读者感受到被理解。

四 · 灵感与突破瓶颈

Rule #9
列出"不会发生什么"
卡住时，列出接下来所有"不会发生"的事。突破素材往往就藏在其中。
Rule #10
拆解你喜欢的作品
你喜欢的东西里藏着你自己。先承认它，才能使用它。90%的故事力来自敏锐的观察力。
Rule #12
排除显而易见的想法
第一个、第二个、第三个——都太明显了。强迫自己想五个，再往下。最好的在后面。

五 · 意义与目的

Rule #14
为什么必须讲这个故事
你内心燃烧的信念是什么，让这个故事有存在的理由？这是故事的心脏。
Rule #19
巧合入困境是智慧，巧合出困境是作弊
用巧合把角色推入麻烦是好的。但用巧合把他们救出来？那是编剧的欺骗。

与今日创作者的关系

对内容创作者

Pixar 法则的内核是换位思考：不是写你想写的，而是写观众需要听的。推文、博客、视频——都是故事，结构从未改变。

Rule #4 的故事骨架可以直接套用到任何pitch、路演或产品介绍：从前（背景）→每天（现状）→有一天（转折）→因为（连锁反应）→最终（结局）。

对创业者和产品人

最好的品牌都是最好的故事。Rule #1（用户爱旅程）是产品迭代的核心心法——让用户成为故事里的角色，你的工具是帮助他们赢的工具。

Rule #14（为什么讲这个故事）是品牌使命声明的底层逻辑：找到你内心燃烧的信念，它会过滤掉所有噪音。
